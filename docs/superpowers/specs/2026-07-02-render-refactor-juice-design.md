# Спека: рендер-рефактор + juice-пас

Дата: 2026-07-02. Статус: утверждена (обсуждение в сессии).

## Цель

Перевести игру из «функционально, но пустовато» в «зрелищно и профессионально»: заменить
rebuild-per-frame рендер на персистентные pooled-объекты, добавить честные снаряды в симуляцию,
провести juice-пас (твины, частицы, screen shake, цифры урона, фильтры) и апгрейд звука.

## Утверждённые решения

| Развилка | Решение |
|---|---|
| Снаряды | **Честные в симе** — урон при долёте; ребаланс через `balance:optimize` |
| Зависимости | Размер не важен: **GSAP (PixiPlugin) + pixi-filters 6.x**; частицы — кастом на нативном Pixi v8 `ParticleContainer` |
| Звук | **WebAudio-синтез остаётся**, серьёзный апгрейд (pitch-рандом, слои, стингеры, адаптивность) |

## Контекст (из инвентаря кода)

- Pixi.js v8 + TS strict + Vite + Vitest; ~8k строк; сим (`src/game`) framework-free, headless-прогоняем (`sim.ts`), есть авто-балансировщик (`scripts/balance-optimize.ts`, полоса fairness [0.15, 0.65] потерянных жизней).
- Критичная перф-проблема: `GameLayers.draw` (`src/render/GameLayers.ts:113-124`) каждый кадр destroy+rebuild один `Graphics` со всеми башнями/врагами/fx.
- Хуки для эффектов уже есть: `game.onSfx(type)` + транзиентный `game.fx[]`.
- Таргетинг O(башни×враги) линейным сканом (`Tower.update`), splash/chain O(враги) на выстрел, spatial-структуры нет.
- `SHOW_DECOR = false` (`src/render/Renderer.ts:23`) — готовая система винтажного декора выключена.
- `RENDER.haloBlur` в палитре мёртв, PIXI-фильтры не используются; «свечение» — полупрозрачные круги.
- Все выстрелы hitscan; `Fx` — косметический луч TTL 0.14s.

## Этап 0 — рендер-рефактор (фундамент)

### 0.1 Event bus

Типизированная шина событий сима вместо строкового `onSfx`:

```ts
type GameEvent =
  | { type: 'shotFired'; kind: TowerKind; from: Vec2; to: Vec2; towerLevel: number }
  | { type: 'enemyDamaged'; enemy: EnemyRef; amount: number; pos: Vec2 }
  | { type: 'enemyDied'; kind: EnemyKind; pos: Vec2; bounty: number }
  | { type: 'leak'; kind: EnemyKind; livesLost: number }
  | { type: 'waveStart'; index: number } | { type: 'waveEnd'; index: number }
  | { type: 'bossSpawn'; pos: Vec2 }
  | { type: 'towerBuilt' | 'towerUpgraded' | 'towerSold'; kind: TowerKind; pos: Vec2 }
  | { type: 'baseHit'; livesLost: number }
  | { type: 'projectileImpact'; kind: TowerKind; pos: Vec2; splash?: number }
```

- Сим публикует; подписчики: рендер (частицы/декали/floating text), звук, screen shake, UI.
- `onSfx` переводится на шину (обёртка для обратной совместимости или полная замена — на усмотрение плана).
- Шина framework-free, живёт в `src/game`, покрыта юнит-тестами.

### 0.2 Персистентные view-объекты (пулы)

- `EnemyView`: контейнер на врага из пула. Текстура глифа запекается 1 раз на тип врага
  (`renderer.generateTexture`) — спрайты батчатся. HP-бар — отдельная мини-графика, обновляется
  на месте. Tint — для белой хит-вспышки. Синхронизация по `enemy.id` (создание/удаление diff-ом).
- `TowerView`: персистентный контейнер на башню; перерисовка только на build/upgrade/boost;
  трансформации (recoil, появление) — твинами, без перерисовки.
- Слои `Renderer` расширяются: `decals` (под врагами), `projectiles`, `particles`
  (нативный `ParticleContainer`), `floatingText` (pooled `BitmapText`), `vfxOverlay` (виньетка, вспышки).
- `GameLayers` (immediate-mode) умирает, заменяется набором view-синхронизаторов.

### 0.3 Spatial grid в симе

- Uniform grid (ячейка ≈ 1 клетка поля) в `src/game`, framework-free, юнит-тесты.
- Использование: таргетинг башен, splash-запросы, chain-запросы теслы, радиус хилера.

### Критерий готовности этапа 0

Игра выглядит и играется как раньше (визуальный паритет), все тесты зелёные, на волне 10
уровня 60×45 нет per-frame аллокаций графики (проверка: DevTools performance, отсутствие
пилообразного GC).

## Этап 1 — честные снаряды

### Модель

- `Projectile` в `src/game`: `{ kind, pos, target, speed, damage, splash?, pierce? }`,
  апдейт в `Game.tick`, урон при долёте.
- **PULSE**: быстрая самонаводящаяся пуля (скорость ~18 клеток/с). Цель умерла в полёте →
  перенаводка на ближайшего врага в радиусе 1.5 клетки, иначе гаснет.
- **MISSILE**: медленная (~7 клеток/с) дуга (визуальная парабола, сим движется по прямой к
  точке упреждения), взрыв в точке прилёта всегда (splash работает по площади). Промах по
  быстрым целям — осознанная фича, контрится SLOW.
- **LASER**: остаётся hitscan (мгновенный луч — физически честно).
- **TESLA**: остаётся мгновенной цепью.
- **SLOW**: аура, без изменений.

### Ребаланс

- Прогон `npm run balance:optimize`; кампания должна остаться в полосе fairness.
- При вылете из полосы — правка статов в `towerTypes.ts` (в первую очередь скорость снарядов,
  упреждение, потом урон/цена).
- Юнит-тесты: попадание, перенаводка, промах ракеты, splash в точке, снаряды при смене скорости игры.

## Этап 2 — juice-пас

### Твин-инфраструктура

- GSAP + PixiPlugin, синхронизация: `gsap.ticker.remove(gsap.updateRoot)` + `gsap.updateRoot(t)`
  из тикера Pixi (уважает паузу/скорость UI-независимо).
- Стандартизованные пресеты easing/длительностей в одном модуле (`src/render/juice/tweens.ts`):
  UI 100–200мс, геймплейные появления 200–400мс easeOutBack.

### Фидбек башен

- Постройка: drop-in squash-stretch (1.3, 0.7)→(0.9, 1.1)→(1,1) ~300мс + кольцо пыли + декаль-паяльная точка.
- Апгрейд: вспышка + краткий скейл; продажа: растворение + частицы.
- Выстрел: recoil-скейл (0.9, 1.1) с пружиной ~120мс.

### Фидбек врагов

- Хит: белая вспышка tint 50мс + визуальный отброс 2–6px против направления выстрела (recovery 100мс).
- Hit-stop ТОЛЬКО на киллах: 40–80мс, босс 120–150мс. Реализация: пауза dt сима, рендер/частицы/UI живут.
- Смерть: разлёт 3–6 осколков (цвет глифа) + вспышка + искры + подпалина-декаль (живёт ~10с,
  затухает; пул с лимитом ~40, старейшие затухают раньше) + floating `+bounty`.
- Дыхание: лёгкий десинхронизированный синус-скейл ±2–3% на врагах.

### Частицы

- Кастомная система поверх нативного `ParticleContainer` v8 (пул, эмиттеры, additive blending).
- Эффекты: muzzle flash, искры попадания (3–8 шт, 200–400мс, разлёт по направлению выстрела),
  трейл ракеты (дым + свечение), взрыв (белая вспышка → расширяющееся кольцо 250мс → 10–30
  осколков → дым), электрическая дуга теслы (midpoint displacement, ветвления ±30°, 2px ядро +
  широкий additive-ореол, перегенерация каждые несколько кадров), поле slow-ауры.
- Глобальный кап ~500 живых частиц, при превышении новые эмиттеры деградируют (меньше частиц).

### Screen shake

- Trauma-система: события добавляют trauma∈[0,1] (утечка +0.3, взрыв ракеты +0.1, смерть босса
  +0.6, удар по базе +0.4), shake = trauma², спад ~1.0/с.
- Perlin/simplex-шум по осям (не random — строб), maxOffset 5–15px, maxAngle 1–3°.
- Трясётся мир (камера), HUD неподвижен. Никогда — на обычные выстрелы.
- Настройка «сниженные эффекты» в настройках + автоуважение `prefers-reduced-motion`
  (отключает shake, hit-stop, RGBSplit, CRT).

### Цифры урона

- Pooled `BitmapText`: спавн ±8px по X, подъём 30–50px за 0.6–1.0с, fade с 50% жизни,
  pop-in 1.4→1.0 за 120мс.
- При плотном огне — агрегация урона по врагу батчами 250мс.
- `+gold` на килл — золотым цветом.

### Фильтры (pixi-filters 6.x)

- AdvancedBloomFilter на слои трасс/снарядов/частиц (на слой, не на объект); fixed `filterArea`;
  на мобиле — 0.5× разрешение bloom.
- RGBSplit 2–4px, затухание 80–150мс — транзиент на смерть босса и удар по базе.
- Деликатный CRTFilter на весь экран (тематично для PCB; выключается «сниженными эффектами»).
- `filters = null` когда слой неактивен.

### Живая плата

- `SHOW_DECOR = true` + перф-проверка (декор статичен → в `cacheAsTexture`/запечённый слой).
- Пакеты-пульсы бегут по копперным трассам (idle-анимация, additive).
- Мигающие LED в декоре с индивидуальной расфазировкой.

### Событийная пунктуация

- Удар по базе: красная edge-виньетка 100–150мс + directional kick камеры от базы + тревога.
- Старт волны: баннер с номером/составом (твин появления).
- Победа: звёзды влетают по очереди со slam-анимацией + фанфары.
- Поражение: тряска + затемнение.

## Этап 3 — звук (синтез-апгрейд)

- Pitch-рандомизация ±1–3 полутона + громкость ±10–15% на всех повторяющихся SFX.
- Бас-слой (короткий thump) под киллы; отдельный жирный — под взрывы ракет.
- Звук для SLOW (сейчас немой): низкий гул-поле при активной ауре.
- Стингеры: старт волны, спавн босса, звезда на экране победы, тревога удара по базе.
- Адаптивная музыка: на босс-волне добавляется слой напряжения (доп. осциллятор/ритм),
  на последней волне — ускорение/плотность.
- Свист ракеты (частотный глиссандо вниз при полёте) — привязан к projectile-событиям.

## Перф-бюджет и ограничения

- 60fps на волне 10 уровня 60×45 (desktop) и стабильные 60 на среднем мобильном на 44×33.
- Частицы ≤ ~500 живых; декали ≤ ~40; пулинг всего (views, частицы, тексты, снаряды).
- Фильтры только на уровне слоёв; максимум 1–2 полноэкранных прохода на мобиле.
- Каждый этап завершается: тесты зелёные, `npm run build` проходит, ручная проверка в браузере.

## Вне скоупа (следующие этапы дорожной карты)

Мета-прогрессия (дерево звёзд, драфт-карты), Heroic/Iron-режимы, тир-визуалы башен,
авторские волны/боссы, новые враги, глобальные способности.

## Порядок работ

Этап 0 → 1 → 2 → 3. Этапы 2 и 3 частично параллелятся (звук независим от визуала).
