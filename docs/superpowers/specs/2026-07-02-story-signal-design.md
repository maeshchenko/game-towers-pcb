# Спека: сюжет «Сигнал извне» (текстовый каркас)

Дата: 2026-07-02. Статус: утверждена (тон: чистый ретро-DOS; объём: текстовый каркас).

## Фабула

19XX. Станция дальней космической связи «ВЕГА-9». Приёмный комплекс ловит повторяющуюся
несущую неизвестного источника. Декодирование показывает: это не сообщение — это программа.
Сигнал самовоспроизводится в аппаратуре станции. Оператор (игрок; без имени, без лица)
изолирует заражённые платы, отступая вглубь комплекса к генератору опорной частоты: если
сигнал захватит генератор — получит передатчик и уйдёт в эфир.

Финальный твист: станция очищена, передача заблокирована — но входящая несущая активна.
Сигнал повторяется. Мы не победили его — мы перестали слушать.

Хоррор через сухость: только системные логи и записи журнала смены. Никаких эмоций в
тексте; чем хуже дела, тем более казённые формулировки. Никаких имён и диалогов.

## Акты

- Акт I — уровни 1–3 (приёмный тракт): процедурная рутина, сигнал локализован.
- Акт II — уровни 4–7 (распространение): аномалии — сигнал адаптируется, копирует оператора.
- Акт III — уровни 8–11 (осада ядра): деградация станции; в логах помехи (глитч-рендер).
- Финал — уровень 12 (генератор): попытка захвата передатчика.

## Подача

1. **POST-интро** — терминальный оверлей перед первым уровнем кампании (и по кнопке «ЛОГ»
   заставки в меню). Печать по буквам, любой клик = мгновенная допечатка, второй клик = закрыть.
2. **Брифинг перед уровнем** — тот же оверлей, «ЖУРНАЛ СМЕНЫ», 3–5 строк. Показывается один
   раз за уровень (флаг в сохранении); повторный просмотр — кнопка «ЛОГ» на карточке уровня.
   При ретрае не показывается.
3. **Дебриф** — 1–2 строки в победном экране + «ОЧИСТКА СТАНЦИИ: NN%».
4. **Финальный лог** — после победы на уровне 12, вместо обычного дебрифа, полноэкранный.
5. **Меню кампании** — подзаголовок «СТАНЦИЯ „ВЕГА-9“ · ОЧИСТКА: NN%»; на карточках
   пройденных уровней статус `ИЗОЛИРОВАНО`, у текущего `ЗАРАЖЕНО`, у закрытых `НЕТ СВЯЗИ`.
6. **Бестиарий** — описания врагов переоформить как «классификацию форм сигнала»
   (номенклатура: ФОРМА-01 «ПАКЕТ» и т.д.); существующие тексты уязвимостей сохранить.

Проценты очистки по уровням (сумма 100): 6, 7, 7, 8, 8, 8, 9, 9, 9, 9, 10, 10.

### Глитч-режим (Акт III)

С уровня 8: в строках брифингов 1–2 случайных символа на строку мерцают подменой
(§, ▓, /, случайная цифра) с периодом ~120мс; исходный символ возвращается. L8 содержит
канонический глитч в самом тексте («Осторо////////но»). Финальный лог глитчит только
строки «СИГНАЛ ПОВТОРЯ…». Глитч отключается тумблером «Сниженные эффекты».

## Техника

- `src/story/campaignStory.ts` — framework-free данные: `{ intro: StoryLine[], final: StoryLine[], levels: Array<{ brief: StoryLine[]; debrief: string; cleanupPct: number }> }`, где `StoryLine = { key: string; glitch?: boolean; pauseMs?: number }`; тексты в i18n (`story.*`), RU+EN.
- `src/ui/StoryScreen.ts` — терминальный оверлей: моноширинный, зелёный на чёрном, курсор-блок, печать по буквам (25–40 мс/символ, пунктуация даёт паузу), клик №1 = допечатать всё, клик №2 = закрыть; глитч-рендер по флагу строки; `prefers-reduced-motion`/`juice.reducedFx` → печать мгновенная, глитч выключен.
- Интеграция `main.ts`: POST-интро перед первым входом в L1 (флаг `storyIntroSeen` в прогрессе); брифинг после выбора уровня до старта (флаг `storyBriefSeen[level]`; при ретрае пропуск); дебриф-строка + процент в `showVictoryScreen`; финальный лог вместо победного экрана L12 (после него — обычный экран с кнопками).
- Меню кампании: подзаголовок с процентом (сумма cleanupPct пройденных), статусы карточек (`ИЗОЛИРОВАНО`/`ЗАРАЖЕНО`/`НЕТ СВЯЗИ`), кнопка «ЛОГ» на пройденных уровнях (повтор брифинга).
- Бестиарий: тексты переоформить под «ФОРМА-NN» номенклатуру (только строки, логика не меняется).
- Тесты: данные истории (12 уровней, проценты в сумме 100, все i18n-ключи существуют в RU и EN); UI — build + ручной QA.

## Вне скоупа

Карта-мир станции, катсцены, озвучка, персонажи. Возможное продолжение твиста — в сиквеле/DLC.
